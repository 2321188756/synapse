# 接口设计

## 1. 概述

Synapse 对外暴露两组接口：

| 接口组 | 用途 | 鉴权方式 |
|--------|------|----------|
| **LLM 兼容接口** | 给聊天客户端调用 | Bearer Token (config.yaml 中的 `api_key`) |
| **管理接口** | 给管理面板 / 命令行调用 | HTTP Basic Auth |

---

## 2. LLM 兼容接口

### 2.1 POST /v1/chat/completions

**说明**：OpenAI 兼容的聊天接口。任何支持 OpenAI API 格式的客户端都可以直接使用。

**请求头**：
```
Authorization: Bearer <api_key>
Content-Type: application/json
```

**请求体**：
```json
{
  "model": "auto",
  "messages": [
    {
      "role": "system",
      "content": "你是一个有用的 AI 助手。"
    },
    {
      "role": "user",
      "content": "帮我搜索 FastAPI 的文档。"
    }
  ],
  "stream": true,
  "temperature": 0.7,
  "max_tokens": 4096
}
```

**请求体字段**：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:--:|--------|------|
| `model` | string | ✅ | - | 模型名称。设为 `auto` 启用智能路由 |
| `messages` | array | ✅ | - | 对话消息数组 |
| `stream` | boolean | ❌ | false | 是否 SSE 流式输出 |
| `temperature` | number | ❌ | 0.7 | 采样温度 |
| `max_tokens` | integer | ❌ | 4096 | 最大输出 token |

**响应 (非流式)**：
```json
{
  "id": "chatcmpl-abc123",
  "object": "chat.completion",
  "created": 1717833600,
  "model": "gpt-4o",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "好的，我来搜索 FastAPI 的文档。\n\n<<<TOOL>>>\nname: web_search\nparams:\n  query: FastAPI documentation\n  engine: tavily\n<<<END>>>"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 200,
    "completion_tokens": 50,
    "total_tokens": 250
  }
}
```

**响应 (流式)**：
```
data: {"id":"chatcmpl-abc123","object":"chat.completion.chunk","choices":[{"index":0,"delta":{"content":"好"},"finish_reason":null}]}

data: {"id":"chatcmpl-abc123","object":"chat.completion.chunk","choices":[{"index":0,"delta":{"content":"的"},"finish_reason":null}]}

...

data: {"id":"chatcmpl-abc123","object":"chat.completion.chunk","choices":[{"index":0,"delta":{"content":""},"finish_reason":"stop"}]}

data: [DONE]
```

### 2.2 GET /v1/models

**说明**：返回可用模型列表。

**响应**：
```json
{
  "object": "list",
  "data": [
    {"id": "auto", "object": "model", "created": 1717833600, "owned_by": "synapse"},
    {"id": "gpt-4o", "object": "model", "created": 1717833600, "owned_by": "openai"},
    {"id": "claude-sonnet-4-6", "object": "model", "created": 1717833600, "owned_by": "anthropic"}
  ]
}
```

### 2.3 POST /v1/messages (Anthropic 兼容)

**说明**：兼容 Anthropic Messages API 格式。Claude Code 等工具可以直接连接。

**请求头**：
```
x-api-key: <api_key>
anthropic-version: 2023-06-01
Content-Type: application/json
```

**请求体**：
```json
{
  "model": "claude-sonnet-4-6",
  "max_tokens": 4096,
  "system": "你是一个有用的 AI 助手。",
  "messages": [
    {"role": "user", "content": "你好"}
  ],
  "stream": true
}
```

**内部处理**：`protocol_bridge.js` 负责将 Anthropic 格式 → 内部标准格式 → 调用 LLM → 转回 Anthropic 格式返回。

## 3. WebSocket 接口

### 3.1 WS /ws

**说明**：WebSocket 连接端点，用于接收服务端推送。

**连接**：
```
ws://localhost:5890/ws?token=<api_key>
```

**客户端 → 服务端消息**：

```json
// 订阅事件类型
{
  "type": "subscribe",
  "events": ["tool_result", "agent_notification", "system_status"]
}
```

**服务端 → 客户端消息**：

```json
// 异步工具执行完成
{
  "type": "tool_result",
  "data": {
    "request_id": "req_abc123",
    "tool_name": "video_gen",
    "status": "success",
    "content": "视频已生成: http://..."
  }
}

// Agent 主动通知
{
  "type": "agent_notification",
  "data": {
    "title": "定时提醒",
    "content": "你设置的'30分钟后提醒我开会'时间到了。",
    "timestamp": "2026-06-08T21:30:00+08:00"
  }
}

// 系统状态变更
{
  "type": "system_status",
  "data": {
    "status": "draining",
    "message": "服务正在重启，请稍候..."
  }
}
```

## 4. 管理接口

所有管理接口需要 HTTP Basic Auth。

### 4.1 GET /api/health

**说明**：健康检查，无需鉴权。

**响应**：
```json
{
  "status": "ok",
  "version": "1.0.0",
  "uptime": 86400,
  "upstream": {
    "status": "ok",
    "provider": "api.openai.com",
    "latency_ms": 120
  }
}
```

### 4.2 GET /api/plugins

**说明**：列出所有已加载的插件。

**响应**：
```json
{
  "plugins": [
    {
      "name": "web_search",
      "display_name": "联网搜索",
      "type": "tool",
      "version": "1.0.0",
      "enabled": true,
      "status": "loaded"
    },
    {
      "name": "weather",
      "display_name": "天气信息",
      "type": "static",
      "version": "1.0.0",
      "enabled": true,
      "status": "loaded"
    }
  ]
}
```

### 4.3 POST /api/plugins/reload

**说明**：热重载所有插件。

**响应**：
```json
{
  "status": "ok",
  "message": "插件已重载",
  "changes": {
    "added": [],
    "removed": [],
    "updated": ["web_search"]
  }
}
```

### 4.4 PUT /api/plugins/:name

**说明**：更新插件配置（启用/禁用、修改配置项）。

```json
{
  "enabled": false,
  "config": {
    "TAVILY_API_KEY": "new-key"
  }
}
```

### 4.5 GET /api/memories

**说明**：列出记忆条目。

**查询参数**：
| 参数 | 说明 |
|------|------|
| `layer` | 记忆层: `l2` / `l3`，默认全部 |
| `tags` | 按标签筛选，逗号分隔 |
| `q` | 关键词搜索 |
| `limit` | 返回条数，默认 50 |
| `offset` | 分页偏移 |

### 4.6 DELETE /api/memories/:id

**说明**：删除指定记忆。

### 4.7 PUT /api/memories/:id

**说明**：编辑指定记忆的标签、重要度等。

### 4.8 POST /api/memories/consolidate

**说明**：手动触发记忆巩固。

### 4.9 GET /api/logs

**说明**：获取最近的服务器日志。

**查询参数**：
| 参数 | 说明 |
|------|------|
| `level` | 日志级别过滤: `debug` / `info` / `warn` / `error` |
| `lines` | 返回行数，默认 100 |
| `requestId` | 按请求 ID 过滤（可选） |

**注**：`/api/logs` 路由内联在 `server.js` 中，非独立路由文件。需要 Basic Auth 鉴权。

**响应**：
```json
{
  "count": 3,
  "filter": { "level": "error", "lines": 100 },
  "logs": [...]
}
```

## 5. 模型路由

### 5.1 智能模型路由

当请求中 `model: "auto"` 时，系统根据对话语义自动选择最合适的模型：

```javascript
// core/llm_client.js — 简化版路由逻辑

const MODEL_ROUTING = {
    fast:     { model: "gpt-4o-mini",     maxTokens: 4096 },   // 简单对话
    default:  { model: "gpt-4o",          maxTokens: 8192 },   // 默认
    coding:   { model: "claude-sonnet-4-6", maxTokens: 8192 }, // 代码任务
    creative: { model: "claude-opus-4-8", maxTokens: 16384 },  // 创作
};

function routeModel(messages) {
    // 简单规则路由 (Phase 1)，后续可接入语义路由
    const lastUserMsg = [...messages].reverse().find(m => m.role === "user")?.content || "";

    const codeKeywords = ["代码", "编程", "bug", "函数", "类", "API", "GitHub"];
    if (codeKeywords.some(kw => lastUserMsg.includes(kw))) return MODEL_ROUTING.coding;

    const creativeKeywords = ["写文章", "故事", "诗歌", "创意", "设计"];
    if (creativeKeywords.some(kw => lastUserMsg.includes(kw))) return MODEL_ROUTING.creative;

    if (lastUserMsg.length < 20) return MODEL_ROUTING.fast;

    return MODEL_ROUTING.default;
}
```

### 5.2 容灾回退

当首选模型不可用时，自动切换到备用模型：

```yaml
# config.yaml
models:
  api_base: "https://api.openai.com/v1"
  api_key: "sk-your-key"
  primary: gpt-4o
  fallback:
    - gpt-4o-mini
    - claude-sonnet-4-6
```

```javascript
// core/llm_client.js — 容灾回退链
async function* chatStreamWithFallback(messages, modelConfig, options) {
    const candidates = [modelConfig.primary, ...(modelConfig.fallback || [])];
    const errors = [];
    for (const c of candidates) {
        try {
            const stream = chatStream(messages, modelConfig, { model: c });
            for await (const chunk of stream) yield chunk;
            return; // 第一个成功即返回
        } catch (e) {
            errors.push(c + ': ' + e.message);
            // 指数退避: 1s, 2s, 3s...
            await new Promise(r => setTimeout(r, 1000 * (errors.length)));
        }
    }
    throw new Error('所有模型均失败: ' + errors.join('; '));
}
```

## 6. 错误码

### 6.1 HTTP 状态码

| 状态码 | 说明 | 场景 |
|:--:|------|------|
| 200 | 成功 | - |
| 400 | 请求格式错误 | messages 为空、model 缺失 |
| 401 | 未授权 | API Key 错误或缺失 |
| 403 | 禁止访问 | 管理接口权限不足 |
| 429 | 频率限制 | 短时间内请求过多 |
| 500 | 服务器内部错误 | 未捕获异常 |
| 503 | 服务不可用 | 重启中 / 上游 API 全部不可用 |

### 6.2 错误响应格式

```json
{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "请求过于频繁，请稍后重试。",
    "details": {
      "retry_after": 30,
      "limit": "60 requests per minute"
    }
  }
}
```

## 8. API 版本策略

- URL 路径不含版本号 (`/v1/chat/completions` 是 OpenAI 的惯例)
- 管理接口以 `/api/` 前缀
- 破坏性变更通过配置开关控制，默认保持兼容
