# 插件系统规范

## 1. 设计哲学

**插件即目录**——加新能力就是新建文件夹 + 写 manifest.yaml + 写代码。

```
添加一个插件:
  mkdir plugins/my_plugin
  写 plugins/my_plugin/manifest.yaml
  写 plugins/my_plugin/main.py
  重启服务 (或热重载)

删除一个插件:
  rm -rf plugins/my_plugin
  重启服务

禁用插件:
  mv plugins/my_plugin plugins/_my_plugin  (下划线前缀 = 禁用)
```

不依赖注册中心、不依赖服务发现、不依赖消息队列——文件系统就是注册中心。

---

## 2. 插件类型

只定义 **3 种** 插件类型，覆盖所有个人使用场景：

| 类型 | 标识 | 说明 | 触发方式 | 典型用途 |
|------|------|------|----------|----------|
| **tool** | `type: tool` | AI 主动调用的工具 | AI 输出 `<<<TOOL>>>` 时触发 | 搜索、生图、文件读写 |
| **static** | `type: static` | 常驻占位符，自动注入上下文 | 每次请求时自动替换占位符 | 天气、时间、系统状态 |
| **preprocessor** | `type: preprocessor` | 消息预处理，在 LLM 调用前修改消息 | 每次请求时自动执行 | 敏感词过滤、上下文折叠 |

**三种类型对比：**

```
请求进来
    │
    ├──→ [所有 preprocessor 插件]  ← 按优先级顺序执行，可修改 messages
    │
    ├──→ [所有 static 插件]        ← 替换 {{占位符}}
    │
    ├──→ LLM 调用
    │
    ├──→ AI 响应中包含 <<<TOOL>>>?
    │       │
    │       ├── 是 → [执行对应 tool 插件] → 结果注入 → 再次 LLM
    │       │
    │       └── 否 → 直接返回给用户
```

## 3. manifest.yaml 规范

每个插件目录下必须有一个 `manifest.yaml`，这是插件与系统之间的唯一契约。

### 3.1 完整示例

```yaml
# plugins/web_search/manifest.yaml

# ===== 基本信息 =====
name: web_search                    # 唯一标识 (snake_case)
display_name: 联网搜索               # 显示名称
version: 1.0.0
description: 使用 Tavily API 搜索互联网信息
author: your-name

# ===== 插件类型 =====
type: tool                          # tool | static | preprocessor

# ===== 运行时 =====
runtime: inline                     # inline (同进程) | subprocess (子进程)
entry: main.py                      # 入口文件
entry_function: execute             # 入口函数名 (默认 execute)

# ===== 启用状态 =====
enabled: true                       # false = 跳过加载

# ===== 工具定义 (仅 type: tool 需要) =====
tool:
  # AI 看到的使用说明 (写在 system prompt 里)
  instruction: |
    使用此工具搜索互联网信息。
    支持的搜索引擎: tavily, google, bing。
    每次搜索返回最多 10 条结果。

  # 调用示例 (AI 学习用)
  examples:
    - description: 搜索 FastAPI 最新文档
      call: |
        <<<TOOL>>>
        name: web_search
        params:
          query: FastAPI latest documentation 2026
          engine: tavily
          max_results: 5
        <<<END>>>

  # 参数定义
  parameters:
    query:
      type: string
      description: 搜索关键词
      required: true
    engine:
      type: string
      description: 搜索引擎
      enum: [tavily, google, bing]
      default: tavily
    max_results:
      type: integer
      description: 最大结果数
      minimum: 1
      maximum: 20
      default: 5

  # 执行配置
  timeout: 30                       # 超时时间 (秒)
  max_retries: 2                    # 失败重试次数

# ===== 占位符定义 (仅 type: static 需要) =====
# static:
#   placeholder: "{{WeatherInfo}}"     # 占位符名称
#   update_interval: 1800              # 更新间隔 (秒)，0 = 每次请求都刷新

# ===== 预处理配置 (仅 type: preprocessor 需要) =====
# preprocessor:
#   priority: 100                      # 优先级 (数字越小越先执行)
#   stage: before_memory              # before_memory | after_memory | before_llm

# ===== 配置项 =====
config:
  TAVILY_API_KEY:
    description: Tavily API 密钥
    required: true
    secret: true                     # secret: true → 日志中自动脱敏
```

### 3.2 字段速查表

| 字段 | 必填 | 类型 | 说明 |
|------|:--:|------|------|
| `name` | ✅ | string | 唯一标识，snake_case，与目录名一致 |
| `display_name` | ✅ | string | 显示名称，管理面板使用 |
| `version` | ✅ | string | 语义化版本 |
| `description` | ✅ | string | 一句话描述 |
| `type` | ✅ | enum | `tool` / `static` / `preprocessor` |
| `runtime` | ❌ | enum | `inline` (默认) / `subprocess` |
| `entry` | ❌ | string | 入口文件名，默认 `main.py` |
| `entry_function` | ❌ | string | 入口函数名，默认 `execute` |
| `enabled` | ❌ | bool | 默认 `true` |
| `tool` | ⚠️ | object | type=tool 时必填 |
| `static` | ⚠️ | object | type=static 时必填 |
| `preprocessor` | ⚠️ | object | type=preprocessor 时必填 |
| `config` | ❌ | object | 插件专属配置项 |

## 4. 插件代码规范

插件通过子进程 (spawn) 运行，通过 stdin/stdout 传递 JSON 通信。主进程使用 Node.js，插件可以用任意语言编写（Python 为主）。

### 4.1 Tool 插件 (type: tool)

```javascript
// plugins/web_search/main.py — Python 插件，spawn 子进程执行
// 通信协议: stdin 读取 JSON 参数 → stdout 输出 JSON 结果

import sys, json

def execute(params, config):
    """插件入口，接收参数，返回 ToolResult"""
    query = params.get("query", "")
    engine = params.get("engine", "tavily")
    max_results = params.get("max_results", 5)

    try:
        results = search_web(query, engine, max_results)  # 实际搜索逻辑
        return {
            "status": "success",
            "content": format_results(results),
            "data": results,
            "message_for_ai": f"搜索完成，共找到 {len(results)} 条结果"
        }
    except Exception as e:
        return {
            "status": "error",
            "error": str(e),
            "message_for_ai": f"搜索失败: {str(e)}"
        }

# stdio JSON 通信: 从 stdin 读参数，向 stdout 写结果
if __name__ == "__main__":
    input_data = json.loads(sys.stdin.read())
    result = execute(input_data["params"], input_data["config"])
    print(json.dumps(result, ensure_ascii=False))
```

```javascript
// core/plugin_executor.js — Node.js 主进程如何调用插件

const { spawn } = require('child_process');

class PluginExecutor {
    async execute(toolCall) {
        const plugin = this.loader.getPlugin(toolCall.name);
        const child = spawn('python3', [plugin.entry], {
            stdio: ['pipe', 'pipe', 'pipe']
        });

        const input = JSON.stringify({
            params: toolCall.params,
            config: plugin.config
        });

        return new Promise((resolve, reject) => {
            let stdout = '', stderr = '';

            child.stdout.on('data', data => stdout += data);
            child.stderr.on('data', data => stderr += data);

            child.on('close', code => {
                if (code !== 0) {
                    resolve({
                        status: 'error',
                        error: stderr || `子进程退出码 ${code}`,
                        message_for_ai: `工具执行失败`
                    });
                    return;
                }
                try {
                    resolve(JSON.parse(stdout));
                } catch (e) {
                    resolve({ status: 'error', error: stdout });
                }
            });

            child.stdin.write(input);
            child.stdin.end();

            // 超时保护
            setTimeout(() => {
                child.kill();
                resolve({ status: 'error', error: '工具执行超时' });
            }, plugin.timeout || 30000);
        });
    }
}
```

**ToolResult 结构 (JSON)**：

```javascript
// 子进程 stdout 输出的 JSON 格式
{
    "status": "success" | "error",
    "content": "",           // 给 AI 看的文本 (必须)
    "data": {},              // (可选) 结构化数据
    "base64": "",            // (可选) Base64 (图片/音频)
    "error": "",             // status=error 时的错误信息
    "message_for_ai": ""     // (可选) 给 AI 的额外提示
}
```

### 4.2 Static 插件 (type: static)

```javascript
// plugins/weather/main.py
import sys, json

def get_value(config):
    city = config.get("WEATHER_CITY", "北京")
    weather = fetch_weather(city)  # 实际获取天气
    return f"{city}天气: {weather.condition}, {weather.temp}°C"

if __name__ == "__main__":
    input_data = json.loads(sys.stdin.read())
    result = get_value(input_data["config"])
    print(result)  // stdout 直接输出字符串 (不是 JSON)
```

**Static 插件 stdout 直接输出字符串**，不是 JSON。

### 4.3 Preprocessor 插件 (type: preprocessor)

```javascript
// plugins/sensitive_filter/main.py
import sys, json

def process(messages, config):
    keywords = config.get("FILTER_KEYWORDS", [])
    for msg in messages:
        if msg["role"] == "user":
            for kw in keywords:
                msg["content"] = msg["content"].replace(kw, "***")
    return messages

if __name__ == "__main__":
    input_data = json.loads(sys.stdin.read())
    result = process(input_data["messages"], input_data["config"])
    print(json.dumps(result, ensure_ascii=False))

## 5. 插件生命周期

```
启动时:
┌──────────┐    ┌──────────────┐    ┌──────────────┐
│ 扫描目录  │ → │ 解析 manifest │ → │ 加载插件代码  │
│ plugins/ │    │ 校验字段      │    │ import 模块   │
└──────────┘    └──────────────┘    └──────┬───────┘
                                           │
                              ┌────────────┴──────────┐
                              │ 成功 → 注册到插件注册表  │
                              │ 失败 → 记录警告，跳过   │
                              └───────────────────────┘

运行时:
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 接收请求      │ → │ 获取工具列表  │ → │ LLM 决定调用  │
│              │    │ (从注册表)   │    │              │
└──────────────┘    └──────────────┘    └──────┬───────┘
                                               │
                              ┌────────────────┴──────────┐
                              │ 找到插件 → 传参 → 执行     │
                              │ 返回 ToolResult           │
                              └───────────────────────────┘

热重载:
POST /api/plugins/reload
  → PluginLoader.reload()
    → 重新扫描 plugins/
    → 对比新旧注册表
    → 新增的插件 → 加载
    → 删除的插件 → 卸载
    → 修改的插件 → 重新加载
```

## 6. 插件配置合并

插件可以有自己的专属配置，优先级从低到高：

```
全局 config.yaml 中的配置
  ↓ 被覆盖
插件 manifest.yaml 中 config 字段的默认值
  ↓ 被覆盖
环境变量 (PLUGIN_<插件名>_<配置项>)
```

**示例**：

```yaml
# config.yaml (全局)
TAVILY_API_KEY: "tvly-global-key"

# plugins/web_search/manifest.yaml (插件)
config:
  TAVILY_API_KEY:
    default: "tvly-plugin-default"
```

```bash
# 环境变量 (最高优先级)
export PLUGIN_WEB_SEARCH_TAVILY_API_KEY="tvly-env-key"
```

**最终生效值**：`"tvly-env-key"` (环境变量优先级最高)

## 7. 插件开发检查清单

开发一个新插件前，依次确认：

- [ ] 确定插件类型：AI 调用的能力 → tool，自动注入的信息 → static，修改消息 → preprocessor
- [ ] 创建目录：`plugins/<snake_case_name>/`
- [ ] 写 `manifest.yaml`：
  - [ ] 基本信息完整 (name, display_name, version, description)
  - [ ] tool 类型：定义 parameters、instruction、examples
  - [ ] static 类型：定义 placeholder
  - [ ] preprocessor 类型：定义 priority、stage
- [ ] 写入口代码：
  - [ ] 函数签名符合规范
  - [ ] 返回类型正确 (ToolResult / str / list[dict])
  - [ ] 异常处理完备
- [ ] 测试：
  - [ ] 正常调用
  - [ ] 参数缺失 (required=True 但 AI 没传)
  - [ ] 超时情况
  - [ ] 上游 API 挂掉
- [ ] 文档：在插件目录下写一个简短的 README

## 9. 安全规则

1. **不要写会修改系统文件的插件**。文件管理插件的 `root_path` 必须限定在 `data/` 或用户指定的安全目录内
2. **不要写会执行任意命令的插件**。如果确实需要 Shell，加白名单和审批流程
3. **敏感配置项标记 `secret: true`**，日志中自动脱敏
4. **所有网络请求插件必须设置 timeout**，默认 30 秒
