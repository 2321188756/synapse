# 部署指南

## 1. 环境要求

| 依赖 | 最低版本 | 说明 |
|------|----------|------|
| Node.js | 20 LTS+ | 主服务运行时 |
| Rust | 1.75+ | 编译向量引擎 (N-API) |
| Python | 3.10+ | 插件子进程运行环境 |
| Docker | 24.0+ | (可选) 容器化部署 |
| 内存 | 512MB | SQLite + USearch 内存映射 |
| 磁盘 | 1GB | 含依赖、向量数据和记忆文件 |

## 2. 本地开发部署

### 2.1 克隆并配置

```bash
git clone https://github.com/your/synapse.git
cd synapse

# 安装 Node.js 依赖
npm install

# 编译 Rust 向量引擎
npm run build:rust

# 安装 Python 插件依赖
pip install -r requirements.txt

# 配置
cp config.example.yaml config.yaml
# 编辑 config.yaml，填入 LLM API Key
```

### 2.2 编辑 config.yaml

```yaml
# config.yaml — 最小可运行配置
server:
  host: "0.0.0.0"
  port: 5890

api_key: "your-secret-key-here"

system_prompt: "你是一个有用的 AI 助手。"

models:
  api_base: "https://api.openai.com/v1"     # OpenAI / NewAPI / Ollama 地址
  api_key: "sk-your-key-here"
  primary: gpt-4o-mini                       # 默认模型名
  fallback:
    - gpt-3.5-turbo
  routing:
    default:  { model: gpt-4o-mini,       max_tokens: 8192 }
    fast:     { model: gpt-3.5-turbo,     max_tokens: 4096 }
    coding:   { model: claude-sonnet-4-6,  max_tokens: 8192 }
    creative: { model: claude-opus-4-8,   max_tokens: 16384 }

admin:
  username: admin
  password: change-me-please
```

### 2.3 启动

**Windows 用户**：双击根目录的 `start.bat`，自动检查环境、安装依赖、启动服务并打开调试页面。

**命令行启动**：

```bash
# 开发模式 (带热重载)
npm run dev

# 生产模式
npm start

# 使用 PM2
npm run pm2:start
npm run pm2:logs
```

## 3. Docker 部署

### 3.1 Dockerfile

```dockerfile
# 多阶段构建: Node.js 主服务 + Rust 向量引擎 + Python 插件
FROM node:20-slim AS node-base

FROM rust:1.75-slim AS rust-builder
WORKDIR /app/rust-vector
COPY rust-vector/ .
RUN cargo build --release

FROM node:20-slim
WORKDIR /app

# 系统依赖 (Python 给插件用)
RUN apt-get update && apt-get install -y --no-install-recommends \
    curl python3 python3-pip \
    && rm -rf /var/lib/apt/lists/*

# Node.js 依赖
COPY package.json package-lock.json ./
RUN npm ci --production

# Rust 编译产物
COPY --from=rust-builder /app/rust-vector/target/release/*.node ./rust-vector/

# Python 插件依赖
COPY requirements.txt .
RUN pip3 install --no-cache-dir -r requirements.txt --break-system-packages

# 应用代码
COPY . .

VOLUME ["/app/data"]
EXPOSE 5890
CMD ["node", "server.js"]
```

### 3.2 docker-compose.yml

```yaml
version: '3.8'

services:
  synapse:
    build: .
    container_name: synapse
    ports:
      - "5890:5890"
    volumes:
      - ./data:/app/data
      - ./config.yaml:/app/config.yaml:ro
      - ./plugins:/app/plugins:ro
    environment:
      - TZ=Asia/Shanghai
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5890/api/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

### 3.3 启动

```bash
docker-compose up -d          # 构建并启动
docker-compose logs -f        # 查看日志
docker-compose down           # 停止
docker-compose down -v        # 完全清理 (含数据卷)
```

## 4. 连接客户端

## 5. 生产环境加固

### 5.1 Nginx 反向代理

```nginx
# /etc/nginx/sites-available/synapse
server {
    listen 443 ssl;
    server_name ai.your-domain.com;

    ssl_certificate     /etc/ssl/certs/your-cert.pem;
    ssl_certificate_key /etc/ssl/private/your-key.pem;

    location / {
        proxy_pass http://127.0.0.1:5890;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        # SSE 流式支持
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }
}
```

### 5.2 systemd 服务

```ini
# /etc/systemd/system/synapse.service

[Unit]
Description=Synapse AI Middleware
After=network.target

[Service]
Type=simple
User=synapse
WorkingDirectory=/opt/synapse
ExecStart=/usr/bin/node /opt/synapse/server.js
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable synapse
sudo systemctl start synapse
sudo systemctl status synapse
```

## 6. 数据备份

### 6.1 自动备份脚本

```bash
#!/bin/bash
# 放到 crontab: 0 3 * * * /opt/synapse/backup.sh

BACKUP_DIR="/backup/synapse"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p "$BACKUP_DIR"

cp /opt/synapse/data/memory.db "$BACKUP_DIR/memory_$DATE.db"
tar -czf "$BACKUP_DIR/vectors_$DATE.tar.gz" /opt/synapse/data/vectors/
cp /opt/synapse/config.yaml "$BACKUP_DIR/config_$DATE.yaml"

# 保留最近 30 天
find "$BACKUP_DIR" -mtime +30 -delete
echo "Backup completed: $DATE"
```

### 6.2 恢复

```bash
cp /backup/synapse/memory_20260608_030000.db /opt/synapse/data/memory.db
tar -xzf /backup/synapse/vectors_20260608_030000.tar.gz -C /opt/synapse/data/
docker-compose restart
```

## 7. 常见问题

### Q: 启动时 Rust 向量引擎报错？

```bash
# 确认 Rust 编译环境
rustc --version  # 需要 ≥ 1.75
npm run build:rust  # 重新编译向量引擎
```

### Q: 数据在哪里？

都在 `data/` 目录：
- `data/memory.db` — 长期记忆 (SQLite)
- `data/vectors/` — 中期记忆向量 (USearch, Rust N-API)

备份这个目录就够了。
