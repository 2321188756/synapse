## 三、发现的问题

### P1 — 建议修复（4 项）

#### 3.1 `routes/models.js` L41 — routing key 误用为 provider

```javascript:41
addModel(entry.model, entry.provider || key);
```

当 `entry.provider` 为空时，`owned_by` 显示为 "fast"/"default"/"coding"/"creative" 等 routing key，而非 "synapse" 或真实 provider 名。

**修复**：
```javascript
addModel(entry.model, entry.provider || 'synapse');
```

#### 3.2 `core/llm_client.js` L139-L145 — Anthropic 错误格式未处理

当前代码将错误响应体当 text 截取前 500 字符，但 Anthropic 错误是 JSON `{type: "error", error: {type: "...", message: "..."}}`。用户看到的错误信息是原始 JSON，可读性差。

**修复**：对 4xx 错误尝试 JSON.parse 后提取 `error.message` 或 `error.error.message`。

#### 3.3 `routes/chat_handler.js` L70 — usage token 始终为 0

非流式响应返回 `usage: { prompt_tokens: 0, completion_tokens: 0, total_tokens: 0 }`。虽然聊天功能不受影响，但某些客户端依赖 usage 统计计费。

**修复**：在 llm_client.js 的流式结束时收集上游返回的 usage（OpenAI 在最后一个 chunk 的 `usage` 字段），或标注 "Phase 3 实现"。

#### 3.4 `CLAUDE.md` L70 — 引用不存在的 §3.9

```markdown
| `modules/logger.js` | `docs/架构设计.md` §3.9 | 日志模块描述、级别规则 |
```

`docs/架构设计.md` §3 只到 3.8（ws_server.js），没有 §3.9。

**修复**：在 `架构设计.md` §3 新增 `§3.9 modules/logger.js — 运行时日志`，或更新 CLAUDE.md 的同步矩阵指向正确的章节。

### P2 — 可延后（4 项）

#### 3.5 `config.example.yaml` 缺少 `system_prompt` 字段

`server.js` L53 支持 `CONFIG.system_prompt` 但配置模板中没有此字段。虽然没有 system_prompt 时走默认值，但配置模板应该文档化所有支持的顶级字段。

**建议**：在 `config.example.yaml` 中增加注释：
```yaml
# ---- System Prompt (可选) ----
# system_prompt: "你是一个有用的 AI 助手。"
```

#### 3.6 `core/llm_client.js` ROUTING 模型名硬编码

L64-L69 的 ROUTING 硬编码了 `gpt-4o`、`claude-sonnet-4-6` 等模型名。如果用户 config.yaml 中配置了不同的模型名，auto 路由可能推荐不存在的模型。

**建议**：Phase 2 时从 `modelConfig.routing` 动态构建 ROUTING，或至少加 `// TODO: Phase 3 从 config 动态读取`。

#### 3.7 `server.js` L11 — `config.env` 文件意图不明确

```javascript
require('dotenv').config({ path: require('path').join(__dirname, 'config.env') });
```

`.gitignore` 中无 `config.env` 排除规则。如果这是为以后的环境变量覆盖预留，应加注释。如果不需要，应删除。

#### 3.8 日志脱敏未实现

架构设计文档 §7 安全考量中写了 "自动脱敏 API Key 和用户内容"，但 `modules/logger.js` 中未实现。当前日志会记录完整的请求信息（虽然不在生产代码中直接打印 API Key，但用户消息可能含敏感内容）。

**建议**：Phase 3（记忆系统涉及用户数据时）实现日志脱敏中间件。
