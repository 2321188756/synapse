## 二、新变更分析

本次变更集中在 `core/memory_engine.js`（22:36 时间戳）。

### 2.1 ✅ `_searchByKeywords()` 方法提取 + 空值保护

L124-135：关键词搜索逻辑从 `recall()` 抽出为独立方法，含空关键词保护。**B2 已修复。**

### 2.2 ✅ 两层级联召回

```
Layer 1: _searchByKeywords(keywords, topK*2) — 关键词 LIKE 匹配
    ↓ 召回不足 topK
Layer 2: _getRecentMemories(topK*2) — 最近/重要记忆兜底
    ↓ seen Set 去重
评分排序 → 取 topK
```

这个设计直接解决了 devlog 中 "memories: 0" 的偶发问题。当用户说 "好"（单字，关键词极少），Layer 1 可能返回空，Layer 2 仍能返回最近记忆。

### 2.3 ✅ `_expandKeywords()` 第一/三人称桥接

L150-156：当关键词含 "我"/"我的" 时补充搜索 "用户"，反之亦然。

**合理性**：AI 以第三人称记录（"用户叫小王"），用户以第一人称提问（"我上次说什么"），关键词不匹配会导致召回失败。这个桥接是正确的。

### 2.4 🟢 桥接词汇覆盖不足

`_expandKeywords` 只处理 "我/我的" ↔ "用户" 一对映射。其他常见的指代差异：
- "自己" / "本人" / "咱" → 无法匹配 "用户"
- "你"（用户对 AI 说 "你上次说..."）→ 不在桥接范围

**建议**：Phase 4 接入 Embedding 向量检索后，语义匹配自然解决此问题，不需要额外处理。

### 2.5 🟢 `_getRecentMemories()` 不区分 layer

L137-144：`SELECT * FROM memories` 按 importance+created_at 排序。Phase 3 所有记忆都是 L3，无影响。Phase 4 引入 L2 后需要加 `WHERE layer = 'l3'` 或根据调用上下文过滤。
